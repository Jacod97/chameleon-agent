# Chameleon Agent

## 1. Overview

### 1.1 배경
모기는 일상생활에서 흔히 접하는 해충으로, 수면 중 소음과 흡혈로 인해 많은 사람들이 불편을 겪고 있다. 기존의 모기 퇴치 방식(스프레이, 모기향 등)은 화학물질 사용에 따른 부작용이 존재하며, 물리적 포획 방식은 자동화가 어렵다.

본 프로젝트는 이 문제를 해결하기 위한 첫 단계로, Unity 엔진 기반 가상환경에서 모기를 자율적으로 포획하는 강화학습 에이전트를 개발한다. 가상환경에서 검증된 정책은 추후 실제 로봇 시스템으로 이식되는 것을 최종 목표로 한다.

### 1.2 목표

Unity 가상환경 속에서 다음 조건을 만족하는 RL 기반 에이전트를 구축한다.

1. 집안에 존재하는 모든 벌레를 포획하는 것을 최우선 목표로 한다.
2. 포획 과정에서 집안 물품을 파손하지 않는다.
3. 추후 실물 로봇 이식을 고려하여, **현실과 동일한 물리 조건이 반영된 환경**에서 구동되는 것을 전제로 설계한다. 즉 환경·에이전트·물체의 동작은 가능한 한 현실을 그대로 반영하며, 게임적 단순화는 피한다.

## 2. Environment

학습 환경은 Unity Engine을 통해 실제 방과 유사한 배치 형태로 구성한 가상 공간을 사용한다. 에이전트는 이 환경과 상호작용하며 학습에 사용할 경험(transition)을 생성한다.

### 2.1 가상 공간 구조

고정된 원룸(7m × 7m × 3m)을 학습 환경으로 사용한다. 벽·바닥·천장으로 둘러싸인 밀폐 공간이며, 집안 물품이 고정된 수와 위치로 배치된다. 방 구조는 매 에피소드마다 동일하다. Unity Physics를 기반으로 중력·충돌 등 현실과 동일한 물리 조건이 적용된다.

### 2.2 카멜레온 에이전트

에이전트는 약 30cm급 소형 홈 로봇 형태로 구현한다.

| 부위 | 구성 |
|------|------|
| 하부 | 바퀴 (구동) |
| 중부 | 몸통 |
| 상부 | 머리: 카메라 + 혀 발사구 |

- 차체는 탱크형 차동 구동 방식으로, 전·후진과 제자리 yaw 회전이 가능하다.
- 머리는 차체와 독립적으로 yaw·pitch 회전이 가능하다.
- 시작 위치는 매 에피소드 방 중앙으로 고정

**모기 감지.** 머리 시점을 기준으로 정의되는 시야 영역 안의 모기를 감지한다. 시야각(FOV)은 약 90°, 유효 감지 거리는 약 3m이며, 가구·벽에 의한 occlusion을 raycast로 판정하여 가려진 모기는 감지되지 않는다. 모기는 크기가 약 5mm로 매우 작아 CNN 기반 영상 인식으로는 안정적으로 검출하기 어렵다고 판단했다. 이에 따라 감지는 카메라 영상(픽셀)을 처리하는 방식이 아니라, 시야·가림 판정을 통과한 모기의 상대 위치·속도를 센서 데이터로 받는 방식으로 구현하며, 이 가변 길이 집합(BufferSensor)을 PointNet으로 인코딩한다. 시야 방향은 머리 회전으로 조절하므로, 모기를 발견하려면 주변을 둘러보는 행동이 필요하다.

**혀(공격 메커니즘).** 실제로 늘어나는 발사체로 모사한다. 최대 사거리는 약 2~3m, 발사·복귀 사이클은 약 0.3초이다. 별도의 쿨다운은 없으며, 사이클 자체가 쿨다운 역할을 한다(사이클 진행 중 추가 발사 입력은 무시). 혀 끝의 흡착력으로 대상을 부착한다.

### 2.3 가구

방 안에는 다양한 가구가 고정된 수와 위치로 배치된다. 각 가구 인스턴스에 현실 무게값(kg)과 파손 임계를 직접 부여한다. 혀가 가구에 접촉하면 흡착이 발생하고, 그 결과는 Unity 강체 물리와 무게값에 의해 자동으로 결정된다.

| 대상 | 결과 |
|------|------|
| 모기 (거의 무중력) | 즉시 빨려옴 → 포획 |
| 가벼운 물체 (음식, 식기, 책, 작은 화분 등) | 흡착되어 끌려오다 혀가 풀린 뒤 바닥에 떨어지며 충돌 |
| 무거운 가구 (책상, 냉장고, 책장 등) | 움직이지 않음. 반작용으로 에이전트가 미끄러지거나 정지 |

가구가 바닥·물체와 충돌할 때 발생한 충격력이 가구별 파손 임계를 넘으면 파손되며, 파손 발생 시 에피소드는 즉시 실패로 종료된다.

### 2.4 모기

모기는 3차원 공간을 자유롭게 비행하는 포획 대상이다.

- 에피소드당 3~10마리가 균등 랜덤으로 생성되며, 초기 스폰 위치는 방 안 임의의 공중 위치이다.
- 랜덤한 방향으로 비행하다 간헐적으로 벽·천장·가구에 착지하여 정지하고, 일정 시간 후 다시 비행한다.
- 카멜레온의 존재를 인식하지 않으므로 회피 행동을 하지 않는다.
- 개체 간 상호작용은 없으며 각각 독립적으로 행동한다.
- 시각·물리 모델은 현실 모기 수준(크기 약 5mm, 비행 속도 약 1m/s)으로 설정한다.
